3. Most important gaps and corrections

Gap 1

Context is the real budget

Best practices mostly follow from one constraint: the context window fills with conversation, file reads, and command outputs. As it fills, quality can drop. The goal is not “give Claude everything,” but “give Claude the smallest sufficient context.”

Gap 2

CLAUDE.md is not a knowledge dump

Use CLAUDE.md for broad, stable, high-value instructions: commands, style rules, workflow expectations, gotchas. Put occasional domain knowledge into skills instead, so it loads only when relevant.

Gap 3

Hooks are stronger than instructions

CLAUDE.md is advisory. Hooks are deterministic. If something must happen every time, such as eslint after edits or blocking writes to migrations, use a hook.

Correction

You still own the feedback loop

Claude can run the agentic loop, but you should still stop it early when direction is wrong, verify success criteria, and reset when the session becomes polluted. Less micromanagement does not mean no steering.

Gap 4

Use subagents to protect main context

For broad investigation, code review, security review, or edge-case hunting, delegate to subagents. They read lots of files in separate context and return compressed findings.

Gap 5

Scale with parallelism

Use worktrees, multiple sessions, non-interactive claude -p, and writer/reviewer patterns. A fresh reviewer session often catches issues because it is not biased by the implementation history.

7. Final cheat sheet

Best-practice prompt template

Goal: [what to build/fix]
Scope: [files/folders/components]
Context: [patterns/docs/errors/screenshots]
Constraints: [no new library, latency, style]
Verification: [test/lint/typecheck/screenshot]
Behavior: explore first if uncertain; otherwise implement directly.

When Claude struggles

After 2 failed corrections: /clear and restart with a sharper prompt.
For broad investigation: use subagents to keep main context clean.
For mandatory behavior: use hooks, not just CLAUDE.md.
For large changes: explore → plan → implement → verify → commit.
For unrelated tasks: /clear before switching topics.

One-line takeaways

Context is the scarce resource; do not waste it with kitchen-sink sessions.
Verification is the highest-leverage input you can give Claude.
CLAUDE.md is for stable rules; skills are for on-demand knowledge; hooks are for guaranteed actions.
Subagents are context firebreaks: let them investigate and report back.
You are not writing every line; you are designing the engineering loop.
